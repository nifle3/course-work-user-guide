# 10. Документация для разработчиков

Раздел описывает внутреннюю логику системы, архитектуру и интеграции.

---

## 10.1 Архитектура приложения

- Fullstack на **Next.js**
- **Clean Architecture**:
  - Presentation Layer — интерфейс на React
  - Service Layer — бизнес-логика (AiService, InterviewService, GamificationService)
  - Data Access Layer — Drizzle ORM для PostgreSQL

- Используется **tRPC** для безопасного и типизированного обмена данными между фронтендом и бэкендом.

---

## 10.2 Интеграция с ИИ

- Используется **LangChain** для управления диалогом и «памятью» нейросети.
- Системные промпты содержат настройки эксперта.
- Ответы анализируются в реальном времени и сохраняются в базе данных.

---

## 10.3 Работа с медиа

- Иконки и аватары хранятся в S3-совместимом хранилище.
- В базе сохраняются только ссылки.

---

## 10.4 Кэширование

- Redis хранит промежуточные состояния сессий и часто запрашиваемые данные для ускорения работы интерфейса.

---

## 10.5 Добавление нового типа критерия

1. Добавить запись в таблицу **ScenarioCriteriaType**
2. Обновить сервис, который формирует системный промпт
3. Протестировать через песочницу
